## Design Notes

### Brand consistency with Commodity Test:
- Same dark background (#0a0a0a)
- Same blue accent (#4d65ff)
- Same Changa font family
- Same score gauge visual treatment
- Same button styles (btn-kinetic, btn-outline)
- Same "problem → solution" page structure

### Key differences from Commodity Test:
- File upload instead of URL input
- Payment wall before processing
- Private results (not public like website analysis)
- More detailed results (full fixes, not just phrases)
- Longer processing time (document analysis vs. page scrape)

### UX priorities:
1. Clear value proposition (before payment)
2. Fast file upload feedback (validation, preview)
3. Transparent pricing (no surprises)
4. Real-time processing updates (reduces anxiety)
5. Actionable results (copy-paste ready)
6. Easy sharing (team collaboration)
